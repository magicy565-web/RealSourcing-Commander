## 第一章：MVP 定义 —— 我们到底在卖什么？

在投入任何开发之前，我们必须清晰地定义产品的最小可行版本（MVP）。这不是功能的堆砌，而是**找到一个能让老板毫不犹豫付费的最小价值闭环**。

### 1.1 产品形态：从“软件”到“解决方案”的认知跃迁

你不应该再把自己定位成一个“SaaS 开发者”，而是“AI 解决方案提供商”。因此，你的产品交付物不是一个 App，而是一个完整的解决方案包。

| 交付物 | 描述 | 价值定位 |
| :--- | :--- | :--- |
| **硬件** | 一台全新的、预装好所有软件的旗舰智能手机（如 iPhone 17 Pro） | **仪式感与专属感**：这不是一个通用的 App，这是专门为你定制的“增长引擎”。 |
| **软件** | 预装并配置好的 Commander App，以及所有必要的第三方 App（如 TikTok, Instagram） | **开箱即用**：老板不需要懂任何技术，拿到手机就能用。 |
| **服务** | 1v1 的上门安装、配置教学，以及一个月的陪跑服务 | **消除焦虑**：你卖的不是工具，是“搞定一切”的承诺。 |

这种“软硬一体”的交付方式，彻底跳出了传统 SaaS“按月付费、随时可停”的模式，进入了“解决方案咨询”的高价值区间。

### 1.2 核心功能：只做一件事，做到极致

MVP 阶段，我们必须抵制“功能蔓延”的诱惑，只做一个功能，但要把它做到不可替代。基于我们之前的讨论，这个功能就是：

**“TikTok 评论区 AI 线索猎手”**

这个功能的用户旅程极其简单：

1.  **连接账号**：老板在“老板手机”上授权他的 TikTok 账号。
2.  **AI 自动监听**：Commander 后台 24 小时自动扫描该账号下所有视频的评论区。
3.  **AI 自动识别**：利用 LLM 分析每一条新评论，识别出包含购买意向的线索（如“多少钱”、“怎么买”、“私个价”）。
4.  **AI 自动回复**：一旦识别出高意向线索，Commander 立即通过 TikTok API 自动发送一条预设好的私信给该用户（例如：“您好，感谢对我们产品的兴趣，我们的销售顾问会尽快与您联系并提供报价！”）。
5.  **老板手机实时通知**：老板的手机会收到一条推送：“新线索捕获！AI 已自动回复用户‘@username’，请销售跟进。”

### 1.3 为什么是这个功能？

- **痛点够痛**：所有做 TikTok 的外贸老板都面临“评论区询盘零散、容易遗漏”的问题。
- **价值够大**：每一个被捕获的线索都可能是一个几万美金的订单，ROI 极高。
- **技术可行**：核心是 LLM 意图识别 + TikTok API，技术上没有爬虫等不可控因素。
- **故事够好**：完美契合“老板手机”的叙事——“你睡觉时，你的 AI 员工在帮你抓订单”。

**结论**：在第一阶段，Commander 的所有研发、营销、销售资源，都应该聚焦在这一个功能上。其他 11 个 Agent，暂时全部砍掉。


## 第二章：技术实现方案 —— 如何用最低成本跑起来

单人创业，技术选型的第一原则是**“能用托管，绝不自建”**。我们的目标不是构建一个完美的系统，而是用最少的代码、最少的运维，让 MVP 跑起来。

### 2.1 技术栈：全面拥抱 Serverless

| 模块 | 技术选型 | 理由 |
| :--- | :--- | :--- |
| **前端** | React Native (Expo) | 你现有的 `CommanderPhone.tsx` 就是基于此，最大化复用代码。 |
| **后端** | Vercel Functions | 无需管理服务器，按需付费，与你现有的 Hono 框架无缝集成。 |
| **数据库** | Supabase (PostgreSQL) | 提供数据库、认证、存储等一体化服务，免费套餐足够启动。 |
| **定时任务** | Vercel Cron Jobs | 简单、可靠，无需自己部署和维护 Cron 服务器。 |
| **AI 模型** | OpenAI API (gpt-4.1-mini) | 用于评论意图识别，成本可控，效果最好。 |
| **TikTok 集成** | TikTok for Business API | 官方 API，稳定可靠，避免了爬虫方案的维护噩梦。 |

这个技术栈的核心优势是**零运维**。你不需要关心服务器、数据库备份、扩容等任何问题，所有精力都聚焦在业务逻辑开发上。

### 2.2 数据流：事件驱动的简单架构

整个业务流程由一个核心事件触发：**TikTok 发送的 Webhook**。

1.  **Webhook 触发**：当你的 TikTok 账号收到新评论时，TikTok 服务器会向你在 Vercel 上部署的一个 API 端点发送一个 POST 请求。
2.  **Vercel Function 处理**：这个 API 端点是一个 Serverless Function，它接收到 Webhook 数据后，立即调用 OpenAI API 进行意图分析。
3.  **意图判断**：如果 LLM 判断该评论是高意向线索，Function 就会做两件事：
    a.  调用 TikTok for Business API 发送自动回复私信。
    b.  将线索信息（评论内容、用户信息、AI 回复等）写入 Supabase 数据库。
4.  **前端轮询**：老板手机上的 Commander App 每 30 秒轮询一次 Supabase 数据库，查询是否有新线索。如果有，就显示在界面上并发出通知。

这个架构极其简单，但非常健壮。它没有复杂的消息队列、没有状态管理，只有一个个独立的、无状态的 Serverless Function。

### 2.3 开发路线图：四周上线

| 周数 | 核心任务 | 交付物 |
| :--- | :--- | :--- |
| **第一周** | **API 接入** | 注册并拿到 TikTok for Business、Supabase、OpenAI 的 API Key，完成 Hello World 调用。 |
| **第二周** | **后端开发** | 开发接收 Webhook 的 Vercel Function，完成 LLM 意图识别和自动回复逻辑。 |
| **第三周** | **前端开发** | 改造 `CommanderPhone.tsx`，接入 Supabase 数据库，展示实时线索列表和 AI 回复状态。 |
| **第四周** | **联调与部署** | 前后端联调，部署到 Vercel 和 App Store / Google Play (TestFlight / Beta)，准备交付。 |

**结论**：这套方案可以在一个月内由一个人完成。关键在于**严格遵守 MVP 原则，不做任何节外生枝的功能**。


## 第三章：商业模式与 GTM 策略 —— 如何收到第一笔钱

好的产品也需要好的商业模式。对于“老板手机”，传统 SaaS 的定价方式是错误的，我们需要一种更符合其“解决方案”定位的模式。

### 3.1 定价：从“卖软件”到“卖解决方案”

**核心原则：按年收费，打包硬件，提供服务。**

| 套餐 | 价格 | 包含内容 | 目标客户 |
| :--- | :--- | :--- | :--- |
| **增长版** | **¥ 49,800 / 年** | 一台 iPhone 17 Pro + Commander 软件授权 + 1v1 上门配置服务 | 刚开始做 TikTok 的外贸老板 |
| **旗舰版** | **¥ 99,800 / 年** | 两台 iPhone 17 Pro Max + Commander 软件授权 + 1v1 上门配置 + 每月一次策略复盘会 | 已经有成熟 TikTok 团队的大卖 |

**为什么是这个价格？**

- **价值锚定**：你的对标不是一个 App，而是一个**月薪 8000 元的销售助理**。一年就是 9.6 万。你的定价只有他的一半，但能 24 小时工作。
- **筛选客户**：这个价格足以过滤掉大量只想“白嫖”或“试试看”的非目标客户，让你能集中精力服务好真正有付费意愿的优质客户。
- **现金流**：年付模式能让你在早期获得健康的现金流，支撑后续的研发和市场投入。

### 3.2 Go-to-Market (GTM) 策略：找到你的前 10 个种子用户

在产品开发的第三周，你就应该开始 GTM。单人创业，没有市场预算，你的核心策略是 **“创始人 IP + 内容营销”**。

1.  **确定你的目标客户画像**：找到一个你最熟悉的垂直行业（比如“假发”、“汽配”、“户外电源”），只做这个行业的客户。
2.  **打造你的个人 IP**：你就是“最懂 TikTok 外贸 AI 的专家”。开始在即刻、小红书、知识星球上分享你的开发过程、你的思考、你对 TikTok 营销的见解。
3.  **内容即产品**：把你正在开发的 Commander App 的截图、录屏，变成你的内容。比如：“今天，我用 AI 自动抓取了 100 条假发行业的 TikTok 评论，发现了 3 个高意向买家。”
4.  **建立私域流量池**：在你的内容中引导用户加你的微信，进入你的“AI 外贸增长交流群”。这里就是你的第一批种子用户池。
5.  **提供免费诊断**：在群里发起活动，免费为 3 个老板的 TikTok 账号做一次“AI 询盘诊断”，把诊断过程和结果公开分享。这就是最好的销售 Demo。
6.  **转化种子用户**：从这些被你帮助过的、认可你专业度的老板中，发展出你的前 10 个付费客户。

**结论**：你的 GTM 策略不是打广告，而是**通过持续输出价值，吸引同频的人**。你的第一个客户，一定是因为信任你这个人，而不是因为你的产品有多完美。

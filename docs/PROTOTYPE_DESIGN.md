# RealSourcing 5.0 Commander — 原型设计文档

> **文档版本：** v1.0  
> **最后更新：** 2026-02-27  
> **原型地址：** `commander-prototype/`（React PWA，基于 Vite + Tailwind CSS 4 + shadcn/ui）  
> **用途：** 本文档记录原型的完整设计决策、已实现模块、交互规范，以及阶段四升级规划，作为产品设计基准文件和后续开发的参考依据。

---

## 1. 产品定位与设计哲学

### 1.1 核心定位

RealSourcing 5.0 Commander 是面向中国外贸中小企业主的**数字秘书型应用**。产品的核心价值主张是：让老板用**最少的操作**驱动一套 AI 系统在海外社媒平台持续工作，而不是让老板学习使用一套复杂的 SaaS 工具。

> "OpenClaw 替你在海外工作，你只需要在早晚两次通知里做决策。"

### 1.2 用户使用模式

产品围绕三种使用场景设计，而非功能导向：

| 使用场景 | 触发方式 | 典型操作时长 | 主要界面 |
|---------|---------|------------|---------|
| **早晚战报** | 飞书定时通知（08:00 / 18:00） | 2-5 分钟 | 手机端今日状态视图 |
| **碎片时间查看** | 主动打开 App | 30 秒 - 2 分钟 | 手机端今日状态视图 |
| **固定时间处理询盘** | 飞书新询盘通知 | 5-15 分钟 | 手机端询盘管理视图 |
| **深度配置** | 主动打开 Web 端 | 15-30 分钟 | Web 管理端 |

### 1.3 设计风格

**Night Commander（暗夜指挥官）**，贯穿手机端和 Web 端的统一视觉语言：

| 设计要素 | 规格 |
|---------|------|
| 主背景色 | `oklch(0.14 0.02 250)` — 深海军蓝 |
| 主强调色 | `#f97316` — 橙色（行动/警告） |
| 成功色 | `#10b981` — 青绿色 |
| 数据色 | `#3b82f6` — 蓝色 |
| AI 色 | `#a855f7` — 紫色 |
| 标题字体 | Space Grotesk |
| 数字字体 | Roboto Mono |
| 设计哲学 | 数据驱动，信息密度高，操作路径短 |

## 2. 原型整体架构

### 2.1 路由结构

```
/                   → 落地页（演示入口）
/phone              → 手机端 Commander（核心演示）
/web                → Web 管理端
/notifications      → 通知中心
/notification-settings → 通知设置
/linkedin           → LinkedIn 托管管理
/facebook           → Facebook 托管管理
/tiktok             → TikTok 托管管理
/whatsapp           → WhatsApp 托管管理
/openclaw           → OpenClaw 数字员工详情
/market             → 新市场开发计划向导
/product-launch     → 新产品发布向导
/geo                → GEO 加速优化向导
/profile            → 个人管理页面
```

### 2.2 全局状态

原型通过两个 React Context 共享全局状态：

**UserPlanContext** — 用户套餐类型（`standard` | `enterprise`），控制功能权限分层展示。标准版用户查看独立部署版专属功能时会看到锁定状态和升级引导。

**NotifSettingsContext** — 通知推送时间设置（小时/分钟），在通知中心、通知设置、手机端状态视图之间共享，倒计时实时更新。

## 3. 手机端 Commander（核心模块）

### 3.1 整体架构

手机端采用**通知驱动架构**，摒弃传统的多 Tab 导航，改为两个主视图的切换：

```
┌─────────────────────────────────┐
│  状态栏（实时时间 + 信号 + 电量）  │
│  顶部导航（Logo + 积分 + 通知 + 头像）│
│  视图切换（今日状态 ↔ 询盘管理）   │
├─────────────────────────────────┤
│                                 │
│  [今日状态视图]                  │
│  或                             │
│  [询盘管理视图]                  │
│                                 │
├─────────────────────────────────┤
│  语音助手浮动按钮（全局）          │
└─────────────────────────────────┘
```

### 3.2 今日状态视图

今日状态视图是老板每天早晚打开 App 的第一个落点，设计目标是**30 秒内获得全局感知**。

**战报卡片区域** 展示四个核心指标：今日新询盘数（含来源分布）、已报价数、跟进中数量、本月成交金额。每个指标卡片带有趋势箭头和环比变化。

**数字员工状态区域** 展示当前 OpenClaw 实例的运行状态。每个实例卡片显示实例名称、运行状态（运行中/暂停/离线）、今日操作次数、待处理消息数、最近一条操作日志。点击实例卡片跳转到 `/openclaw` 详情页。

**待处理事项区域** 以优先级排序展示需要老板处理的事项，包括新询盘提醒（带置信度标签）、跟进到期提醒、账号健康预警。每条事项右侧有快捷操作按钮，点击跳转到对应处理页面。

**下次推送倒计时** 显示距离下次飞书定时推送的剩余时间（实时更新，每分钟刷新）。

**发起任务入口** 底部固定的「+ 发起任务」按钮，点击弹出任务类型选择弹窗，包含五大场景的快捷入口：新市场开发、新产品发布、GEO 优化、社媒策略调整（引导至 Web 端）、OpenClaw 配置（引导至 Web 端）。

### 3.3 询盘管理视图

询盘管理视图是处理具体业务的核心界面，支持完整的询盘生命周期管理。

**三 Tab 分类：**

| Tab | 包含状态 | 典型操作 |
|-----|---------|---------|
| 待处理 | 未回复、未报价 | 查看详情、发起报价 |
| 跟进中 | 已报价、客户未回复 | 查看跟进时间线、手动催单 |
| 已结束 | 已转出、已转合同、已失效 | 查看历史记录 |

**三维筛选系统：** 点击筛选按钮弹出底部抽屉，支持按渠道（Alibaba/LinkedIn/Facebook/TikTok/WhatsApp/SEO/GEO）、买家国家（越南/德国/美国等）、预估金额范围（< $1K / $1K-$10K / $10K-$100K / > $100K）三个维度组合筛选。已激活的筛选条件在按钮上显示数量角标。

**询盘卡片** 展示：渠道图标、公司名称、国家旗帜、产品名称、预估金额、置信度标签（高意向/中等/待验证）、状态标签、接收时间。

### 3.4 询盘详情与报价流程

点击询盘卡片进入详情页，采用**步骤式流程设计**，分为四个步骤：

**步骤一：询盘概览（overview）**

展示买家完整信息（公司/国家/职位/邮件/WhatsApp）、询盘原始内容（英文）、AI 分析摘要、三维置信度评分（渠道权重/内容质量/买家完整度，每项 0-100 分，总分加权计算）。

置信度评分视觉化：三个进度条分别展示三个维度，总分以大字号数字 + 颜色标签（绿色高意向 ≥ 80 / 橙色中等 50-79 / 灰色待验证 < 50）呈现。

底部操作区：「发起报价」（主操作，橙色）、「转人工」（次操作）、「标记失效」（危险操作，需确认）。

**步骤二：报价填写（quote）**

从飞书多维表格拉取行业报价模板（原型中模拟加载动画），展示模板字段：产品名称（预填）、单价、货币、计量单位、价格条款（FOB/CIF/EXW）、最小起订量、交货期、报价有效期。

AI 草稿区域展示中英文双语草稿，支持直接编辑。语音输入按钮可触发语音助手填写价格。

跟进风格选择：三种风格卡片（强势/友好/商务），每种显示风格描述和预览话术。

**步骤三：预览确认（preview）**

展示完整的英文回复内容（含报价详情），支持最终编辑。显示将要执行的操作：通过 [渠道] 发送回复 + 写入飞书多维表格 + 24h 后自动跟进。

「确认发送」按钮触发模拟发送动画，成功后询盘状态更新为「已报价」，自动跳回询盘列表。

**步骤四：转人工（transfer）**

输入业务员姓名和转交备注，确认后询盘状态更新为「已转出」，显示转出信息。

### 3.5 已报价询盘的跟进时间线

状态为「已报价」或「客户未回复」的询盘详情页，在概览步骤下方展示**跟进时间线**：

报价摘要卡片显示：报价金额、跟进风格、下次跟进倒计时（实时更新）。

时间线节点包含：报价发送节点（含发送时间）、已执行的跟进记录（含已读/未读/已回复状态）、待执行的下次跟进节点（含计划时间）。

### 3.6 语音助手

全局浮动麦克风按钮（右下角固定），点击弹出语音助手面板。

**安全操作**（直接执行，无需二次确认）：查询询盘数量、查看战报、切换视图。

**敏感操作**（显示预览，需人工确认后执行）：发送报价、回复买家、标记状态。

原型中内置 8 条示例语音指令，点击可模拟语音识别结果，展示操作预览或直接执行效果。

## 4. 社媒账号托管模块（阶段二）

四个平台的托管管理页面共享统一的设计语言，但各自针对平台特性做了差异化设计。

### 4.1 LinkedIn 托管管理（`/linkedin`）

LinkedIn 是 B2B 外贸的核心主动开发渠道，页面设计围绕**账号健康保护**和**精准外联**两个核心目标。

**账号健康面板** 展示：健康评分（0-100）、今日连接请求使用量（14/20）、InMail 使用量（6/10）、代理节点信息（SG-Residential-07）、账号名称。进度条颜色随使用率变化（绿→橙→红）。

**连接请求管理** 展示待处理的连接请求列表，每条显示：买家姓名/职位/公司/国家、AI 意向评分（0-100）、公司规模、共同联系人数。操作按钮：接受（绿色）、忽略（灰色）、转为询盘（橙色，仅高意向显示）。

**InMail 私信管理** 展示收件箱中的 InMail 消息，每条显示：发件人信息、消息摘要（截断显示）、接收时间、意向评分。点击展开全文，显示 AI 生成的回复草稿，支持编辑后发送。

**账号指标** 展示：个人主页浏览量、搜索出现次数，辅助判断账号活跃度。

### 4.2 Facebook 托管管理（`/facebook`）

Facebook 定位为**个人主页私信开发**渠道，与 LinkedIn 的区别在于更强调私信自动回复能力。

**账号健康监控** 显示：健康状态（健康/警告/受限）、今日操作额度（已用/上限）、代理节点。

**私信对话列表** 展示 Messenger 私信，每条显示：发件人头像/姓名、消息预览、接收时间、意向类型（询盘/连接/其他）、意向分数。

**自动回复开关** 私信可开启全自动回复（AI 直接发送），评论回复需人工确认。开关状态在页面顶部明显展示。

**评论管理** 展示帖子评论中的询价评论，支持批量回复或转为询盘。

**公司主页托管** 标注为「待开发」状态，展示功能预告。

### 4.3 TikTok 企业账号托管（`/tiktok`）

TikTok 定位为**内容驱动的被动获客**渠道，页面重点展示内容表现数据和评论转化。

**内容表现看板** 展示近期发布视频的核心指标：播放量、完播率、互动率（点赞+评论+分享）、涨粉数。数据以卡片形式展示，带趋势箭头。

**高意向评论识别** 系统自动识别含 "how much" / "price" / "MOQ" / "wholesale" 等关键词的评论，按意向分数排序展示。每条评论显示：用户名、评论内容（关键词高亮）、意向分数、所属视频标题。

**评论回复流程** 点击评论进入回复界面，展示 AI 生成的回复草稿（英文），支持编辑后确认发送。高意向评论可一键「转为询盘」，跳转到询盘管理流程。

**账号指标** 展示：粉丝数、总播放量、平均互动率、本月新增询盘数（来自评论转化）。

### 4.4 WhatsApp Business 托管（`/whatsapp`）

WhatsApp 定位为**高转化率的直接沟通**渠道，页面围绕对话管理和快捷回复设计。

**对话列表** 展示所有活跃对话，每条显示：联系人名称/国家、最后一条消息预览、接收时间、未读数量、意向标签。

**快捷回复模板** 底部固定的快捷回复区域，预置常用话术（问候语/索取规格/发送目录/报价跟进），点击直接插入输入框。

**自动回复开关** 可开启非工作时间自动回复（发送预设的「我们已收到您的消息」模板）。

**转为询盘** 对话中识别到询价意图时，显示「转为询盘」按钮，点击自动提取对话中的产品/数量信息，创建询盘记录。

## 5. 任务发起向导模块（阶段三）

### 5.1 OpenClaw 数字员工详情（`/openclaw`）

展示单个 OpenClaw 实例的完整状态，分三个 Tab：

**账号详情 Tab** 展示该实例托管的所有社媒账号，每个账号卡片显示：平台图标、账号名称、今日操作次数、待处理消息数、最后活跃时间、账号状态。

**操作日志 Tab** 时间线形式展示最近 50 条操作记录，每条显示：时间戳、操作描述（平台 + 动作 + 对象）、结果（成功/失败/跳过）。支持按平台筛选。

**资源监控 Tab** 展示 ECS 实例的资源使用情况：CPU 使用率、内存使用率、网络延迟、在线率，以进度条和数字形式展示。

页面顶部显示实例基本信息：实例名称、部署区域（新加坡 SG1）、运行时长、今日总操作次数。

### 5.2 新市场开发计划向导（`/market`）

五步向导流程（`market → product → strategy → budget → review`）：

| 步骤 | 内容 |
|------|------|
| 目标市场 | 选择目标国家/地区（多选，预置越南/德国/美国等热门市场） |
| 推广产品 | 选择要推广的产品（从产品库选择或新建） |
| 开拓策略 | 选择 Agent 组合（调研 Agent + 内容 Agent + 外联 Agent） |
| 积分预算 | 设置本次计划的积分预算上限，展示预计操作次数 |
| 确认计划 | 汇总展示所有配置，确认启动 |

**进行中的计划** 页面上方展示已运行的市场开发计划卡片，每张卡片显示：计划名称、目标市场、进度（已完成 Agent 任务数/总数）、各 Agent 状态（调研中/内容生成中/外联中）、本周新增询盘数。

### 5.3 新产品发布向导（`/product-launch`）

五步向导流程（`info → images → generate → preview → publish`）：

| 步骤 | 内容 |
|------|------|
| 产品信息 | 填写产品名称、核心卖点（多条）、目标市场、行业分类 |
| 产品图片 | 上传产品图片（支持多张，展示上传预览） |
| AI 生成 | 触发 AI 内容生成（模拟进度动画），生成多语言多平台内容 |
| 内容预览 | 分平台展示生成的内容（LinkedIn 帖子/Facebook 帖子/TikTok 视频脚本），支持编辑 |
| 发布确认 | 选择发布平台（多选），确认发布时间，提交 |

**已发布产品** 页面下方展示历史发布记录，每条显示：产品名称、发布平台数量、发布时间、各平台互动数据汇总。

### 5.4 GEO 加速优化向导（`/geo`）

四步向导流程（`engines → keywords → schedule → confirm`）：

| 步骤 | 内容 |
|------|------|
| 选 AI 引擎 | 多选目标 AI 搜索引擎（Perplexity / ChatGPT / Gemini / Claude / Grok / You.com） |
| 产品关键词 | 输入核心关键词（产品名 + 行业词 + 场景词），AI 辅助扩展 |
| 优化周期 | 选择持续优化周期（1个月/3个月/6个月），展示预计效果 |
| 确认启动 | 汇总配置，展示积分消耗预估，确认启动 |

**进行中的 GEO 任务** 展示已运行的 GEO 优化任务，每条显示：目标引擎列表、关键词数量、已运行天数、引用次数（被 AI 引用的次数）、曝光量（AI 回复中出现的次数）。

## 6. Web 管理端（`/web`）

### 6.1 导航结构

左侧边栏（可折叠，56px 收起 / 224px 展开），七个导航模块：

| 模块 | 图标 | 主要内容 |
|------|------|---------|
| 增长总览 | LayoutDashboard | 询盘趋势图、市场分布、近期询盘列表 |
| 询盘管理 | MessageSquare | 询盘列表（含状态统计）、近期询盘详情 |
| Agent 任务 | Target | 任务列表（运行中/已完成/等待中）、任务详情 |
| OpenClaw 管理 | Monitor | 实例列表、账号详情、操作日志、资源监控 |
| 数字资产库 | Database | 客户联系人/沟通记录/产品档案/市场情报/交易历史/买家评价 |
| GEO 监控 | Globe | GEO 任务状态、引用数据、关键词排名 |
| 通知中心 | Bell | 通知历史、推送统计、渠道状态 |

### 6.2 增长总览

**核心指标卡片行**：今日新询盘（+3 vs 昨日）、本月询盘总量（+28%）、本月报价金额、OpenClaw 今日操作次数。

**询盘趋势图**：6 个月的月度询盘量折线图，区分 RFQ 渠道（橙色）和 GEO 引流（紫色）两条线，使用 Recharts AreaChart 实现。

**市场分布柱状图**：按国家展示询盘量分布（越南/德国/美国/日本/澳洲），使用 Recharts BarChart 实现。

**近期询盘列表**：展示最新 5 条询盘，含公司名/国家/产品/预估金额/来源渠道/状态/时间。

## 10. 交互规范

### 10.1 核心交互原则

**30 秒决策原则**：任何需要老板做决策的操作，从看到通知到完成操作不超过 30 秒。报价流程中的每个步骤都有默认值，老板可以不修改直接点「下一步」。

**确认前可撤回**：所有涉及发送消息的操作，在最终确认前都可以返回修改。发送后的操作不可撤回，但会在通知中心留有记录。

**状态可感知**：OpenClaw 的每次操作都会在日志中留有记录，老板可以随时查看「AI 在帮我做什么」。

### 10.2 手机端操作规范

| 操作类型 | 交互方式 | 反馈方式 |
|---------|---------|---------|
| 主要操作（发起报价/确认发送） | 大按钮，橙色，底部固定 | 加载动画 + 成功 Toast |
| 次要操作（转人工/标记失效） | 小按钮，灰色，内联 | 确认弹窗 + Toast |
| 危险操作（删除/归档） | 需要二次确认 | 确认弹窗（红色警告） |
| 列表滑动 | 上下滚动，无分页 | 加载更多（触底加载） |
| 返回 | 左上角返回按钮 | 无动画（保持简洁） |

### 10.3 颜色语义规范

| 颜色 | 语义 | 使用场景 |
|------|------|---------|
| 橙色 `#f97316` | 行动/待处理 | 主操作按钮、新询盘标签、积分显示 |
| 青绿 `#10b981` | 成功/正常 | 成功状态、高意向标签、已完成 |
| 蓝色 `#3b82f6` | 数据/信息 | 数据图表、已报价状态、信息提示 |
| 紫色 `#a855f7` | AI/智能 | AI 生成内容、GEO 相关、AI 分析 |
| 红色 `#ef4444` | 危险/错误 | 危险操作、错误提示、账号受限 |
| 灰色 `#94a3b8` | 次要/禁用 | 次要信息、禁用状态、待验证 |

## 11. Mock 数据规范

原型中所有数据均为 mock 数据，设计时遵循以下原则：

**真实感原则**：买家公司名、国家、联系人姓名均使用真实感强的名称（非 "Test Company"），金额范围符合实际外贸场景（$50 - $200K）。

**多样性原则**：询盘数据覆盖 9 个来源渠道、8 个国家、7 种状态，确保演示时能展示系统的全面性。

**一致性原则**：同一买家在不同模块中出现时，信息保持一致（如 SunPower Solutions 在询盘列表和 Web 管理端都出现）。

**边界情况覆盖**：包含高置信度（87-94分）、中等置信度（68-72分）、低置信度（31-38分）三类询盘，以及已转出、已转合同等终态询盘。
